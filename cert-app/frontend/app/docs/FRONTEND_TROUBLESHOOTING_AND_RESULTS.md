# 프론트엔드: 전·후 차이, 트러블슈팅, 성과 요약

---

## 1. React Query만 도입했을 때 전·후 지연 차이 (구체적 수치)

- **범위**: 다른 변경 없이 **React Query 도입만** 적용했을 때의 효과 (캐시·staleTime에 따른 예상치).
- **전**: 동일 데이터를 쓸 때마다 매번 `fetch` 호출.
- **후**: `staleTime` 내 동일 `queryKey`면 캐시 사용 → **재요청 시 네트워크 지연 0ms**.

### 1-1. 백엔드 1회 응답 R = 500ms 가정 시 (구체적 수치)

| 시나리오 | 전 요청 수 | 후 요청 수 | 전 체감 지연 | 후 체감 지연 | 감소 지연 | 감소율 |
|----------|------------|------------|--------------|--------------|-----------|--------|
| 같은 전공 2번 선택 (추천) | 2회 | 1회 | **1,000ms** | **500ms** | **500ms** | **50%** |
| 같은 필터로 목록 2번 진입 | 4회 | 2회 | **2,000ms** | **1,000ms** | **1,000ms** | **50%** |
| 같은 자격증 상세 2번 방문 | 4회 | 2회 | **2,000ms** | **1,000ms** | **1,000ms** | **50%** |
| 전공 목록 3곳 사용 (추천·마이·메뉴) | 3회 | 1회 | **1,500ms** | **500ms** | **1,000ms** | **67%** |

### 1-2. 백엔드 1회 응답 R = 300ms 가정 시 (구체적 수치)

| 시나리오 | 전 체감 지연 | 후 체감 지연 | 감소 지연 | 감소율 |
|----------|--------------|--------------|-----------|--------|
| 같은 전공 2번 선택 | **600ms** | **300ms** | **300ms** | **50%** |
| 같은 필터로 목록 2번 진입 | **1,200ms** | **600ms** | **600ms** | **50%** |
| 같은 상세 2번 방문 | **1,200ms** | **600ms** | **600ms** | **50%** |
| 전공 목록 3곳 사용 | **900ms** | **300ms** | **600ms** | **67%** |

### 1-3. 요약 (React Query만 사용했을 때)

- 위 수치는 **React Query의 캐시 동작만**으로 얻는 효과 기준 **예상치** (실측 A/B 아님).
- 재방문·동일 파라미터 재요청 시: **요청 횟수·체감 지연 모두 약 50% 감소**, 전공 목록처럼 여러 화면에서 쓰는 데이터는 **약 67% 감소**.
- R=500ms일 때, 예: 같은 전공 2번 선택 시 **1,000ms → 500ms (500ms 절감)**.

## 2. 프론트엔드 트러블슈팅

### 2-1. 청크 로드 실패 (ChunkLoadError)

- **증상**: 배포 후 사용자가 예전 캐시로 접속 시 `Failed to fetch dynamically imported module` / `Loading chunk ... failed`.
- **원인**: Vite 빌드로 청크 해시가 바뀌었는데, 브라우저가 예전 `*.js` 청크를 요청해 404 발생.
- **대응**:
  - `ErrorBoundary`에서 해당 에러를 감지 → **ChunkLoadError** 전용 UI 노출.
  - "페이지를 불러오지 못했습니다. 배포 업데이트 후 발생할 수 있습니다." 문구 + **새로고침** 버튼 제공.
  - 콘솔에 강제 새로고침(Ctrl+Shift+R) 또는 캐시 삭제 안내.
- **파일**: `src/components/ChunkLoadError.tsx`, `src/components/ErrorBoundary.tsx`.

### 2-2. API 타임아웃·일시 오류

- **증상**: 백엔드 지연·5xx·네트워크 끊김 시 사용자 화면에서 요청 실패.
- **대응**:
  - **타임아웃**: 일반 API 15초, AI 추천 45초 (`api.ts` `timeoutMs`).
  - **재시도**: 네트워크/5xx만 2회 재시도, 지수 백오프(1초, 2초). 4xx는 재시도 안 함.
  - **에러 메시지**: 서버 `detail`/`message` 파싱 후 `toast` 등으로 노출.
- **React Query**: 전역 `retry: 1` (main.tsx), 개별 쿼리는 `staleTime` 위주로 캐시 활용.

### 2-3. 인증(OTP) 검증 실패 시 fallback

- **증상**: Supabase `signInWithOtp` 후 `verifyOtp` 시 타입 불일치 등으로 실패.
- **대응**: UserMenu 등에서 `verifyOtp` 실패 시 **`signup` 타입으로 한 번 더 시도** 후 실패 시 에러 throw.
- **파일**: `src/components/layout/UserMenu.tsx`.

### 2-4. 라우팅·포커스 (접근성)

- **증상**: SPA에서 라우트만 바꾸면 스크린리더/키보드 사용자가 "페이지가 바뀌었는지" 인지하기 어려움.
- **대응**: 라우트 변경 시 `#main-content` 내 **h1 또는 `[data-main-heading]`**으로 포커스 이동 (`App.tsx`). 스킵 링크는 `:focus-visible`일 때만 표시.

### 2-5. CORS·환경 변수

- **증상**: 로컬에서 백엔드 호출 시 CORS 에러 또는 잘못된 API URL.
- **대응**: `VITE_API_BASE_URL`로 백엔드 주소 설정. 배포(Vercel)와 로컬 분리. CORS는 백엔드에서 허용 도메인 관리.

## 3. 프론트엔드 성과 (내세울 수 있는 결과)

### 3-1. 체감 지연·요청 수 감소

- React Query 도입으로 **재방문·동일 조건 재요청 시** 네트워크 요청 및 체감 지연 **약 50% 수준 감소** (상세 수치는 §1 참고).
- 전공 목록 등 공통 데이터는 **한 번 불러오면 10분간 재사용** → 중복 요청 제거.

### 3-2. AI 추천 UX·가시화

- **AI 추천 페이지**에서 RAG 파이프라인(3-way Hybrid, 융합+Reranker, Recall·MRR) 및 베이스라인 대비 성능 개선 수치를 **카드 형태로 노출** → “검색 품질을 보여주는 진입점” 제공.
- 전공·관심사 입력 후 **sessionStorage 캐시**로 뒤로가기 시 재요청 없이 결과 복원.

### 3-3. 접근성·SEO 친화 라우팅

- **시맨틱 레이아웃**(header / nav / main / footer)과 **스킵 링크·본문 포커스**로 키보드·스크린리더 사용자가 페이지 구조 인식 및 본문으로 바로 이동 가능.
- 라우트별 **document.title** 설정으로 타이틀 일관성·SEO 대응.

### 3-4. API 계층 안정성

- **용도별 타임아웃**(일반 15초, AI 45초), **5xx/네트워크 재시도**, **서버 에러 메시지 파싱 노출**로 무거운 AI API도 프론트에서 안정적으로 처리.

### 3-5. 에러 복구 UX

- **ErrorBoundary**로 런타임 오류 시 전체 크래시 대신 폴백 UI + 새로고침 유도.
- **ChunkLoadError** 전용 화면으로 배포 후 캐시 이슈 시 사용자가 원인을 이해하고 새로고침할 수 있도록 유도.

### 3-6. 마이페이지·홈 최적화

- **마이페이지**: 취득 자격증 기반 **XP·레벨·티어**(Bronze→Diamond) 로컬 계산, 5분 TTL 로컬 캐시로 관심/최근본/전공별 추천 부담 완화.
- **홈**: 인기 자격증 영역 **뷰포트 진입 시에만** API 호출(IntersectionObserver) → 초기 로딩 비용 절감.

## 4. 한 줄 요약

- **전·후**: 재방문·동일 조건 시 요청·체감 지연 **약 50% 수준 감소**.
- **트러블슈팅**: 청크 로드 실패 전용 UI, API 타임아웃·재시도·에러 노출, OTP fallback, 라우팅 포커스·CORS 대응.
- **성과**: React Query 지연 감소, AI 추천 파이프라인 가시화, 시맨틱·접근성·SEO, API 안정성, 에러 복구 UX, 마이페이지·홈 최적화.
