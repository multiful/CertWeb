# Enhanced RAG 고도화 가이드

## 1. 벡터 vs 키워드: 어디를 고도화해야 할지 판단하기

### 진단 방법

- **질의 유형별로 골든셋을 나눠 Recall/MRR을 비교**
  - **키워드형**: 짧은 질의(2단어 이하), "기사/산업기사/기능사", "SQL", "컴퓨터" 등 → 주로 **BM25(키워드)** 영향이 큼.
  - **자연어/장문형**: 문장 형태, "~하는 자격증 추천해줘", "데이터 분석 관련" 등 → **벡터(임베딩)** 영향이 큼.
- **실행 예**
  - 골든셋을 "키워드형 / 자연어형" 두 그룹으로 라벨링하거나, 질의 길이(단어 수)로 구간 나눔.
  - 동일 파이프라인으로 그룹별 Recall@5, MRR을 측정.
- **해석**
  - 키워드형 그룹에서 지표가 낮으면 → **키워드(스파스) 고도화** 우선: BM25 필드 확장, 동의어, 토크나이저, 인덱스 필드(name, summary 등).
  - 자연어형 그룹에서 지표가 낮으면 → **벡터(임베딩) 고도화** 우선: 임베딩 모델, 청킹, 도메인 파인튜닝.

### 빠르게 확인하는 방법

- **실패 케이스 샘플 보기**
  - 상위 4개에 정답이 안 들어온 질의만 골라서, 그 질의가 "자격증명/코드명 위주"인지 "문장/설명 위주"인지 구분.
  - 전자 비중이 크면 키워드 쪽, 후자 비중이 크면 벡터 쪽을 먼저 손보면 됨.
- **RRF 전 단계 점수 활용**
  - BM25만 켠 검색 vs 벡터만 켠 검색을 각각 돌려서, 어떤 쪽에서 정답이 아예 상위 20 안에 안 들어오는지 보면, 그 쪽이 병목.

## 2. 최고의 RAG를 만들기 위한 발전 순서 제안

### 2.1 검색(Retrieval) 품질

| 우선순위 | 영역 | 구체적 방향 |
|----------|------|-------------|
| 1 | **진단 고정** | 골든셋 2개 이상(revise, vs_baseline 등)으로 질의 유형별 Recall/MRR 측정. 벡터 vs 키워드 병목 파악. |
| 2 | **청킹** | 의미 단위 청킹(semantic chunking), 적당한 overlap, 자격증명/요약이 한 청크에 묶이도록. |
| 3 | **벡터** | 도메인 데이터로 임베딩 파인튜닝(예: 자격증 Q-A 쌍), 또는 더 나은 다국어/한국어 베이스 모델. |
| 4 | **키워드** | BM25 인덱스 필드 확장(name, summary, keyword 컬럼), 동의어/약어 사전, 오타 허용(엘라스틱 등). |
| 5 | **리랭커** | 현재 cross-encoder 유지. 필요 시 골든셋으로 cross-encoder 파인튜닝 또는 2단계(1단계 경량 → 2단계 정밀). |

### 2.2 지연/운영

- **Gating**: 이미 적용됨. RRF 상위가 확실할 때 리랭커 스킵 → 쉬운 질의에서 지연 감소.
- **캐시**: 동일 질의 재요청 시 Redis 등으로 검색 결과 캐시.
- **원격 리랭커(HF Space)**: ONNX/경량화는 Space 쪽 앱에서 적용. 이 백엔드는 URL 호출만 함.

### 2.3 생성(Generation) 품질

- **RAGAS 등으로 분리 측정**: context_precision/recall(검색), faithfulness/answer_relevancy(생성).
- 검색이 좋은데 답변 품질이 낮으면 → 프롬프트, 시스템 메시지, 최종 top_k 조정.
- 검색이 약하면 → 위 Retrieval 고도화를 먼저.

### 2.4 평가 루프

- **다중 골든셋**: revise, vs_baseline, 도메인별 세트 등으로 회귀 방지.
- **질의 유형별 메트릭**: 키워드형/자연어형 그룹별로 지표를 보면 다음 개선 포인트가 명확해짐.

---

## 3. 요약

- **벡터 vs 키워드**: 질의 유형별(키워드형 / 자연어형) Recall·MRR을 나눠 보면, 어느 쪽을 고도화할지 결정할 수 있음.
- **최고의 RAG**: 진단(질의 유형별 지표) → 청킹·벡터·키워드 순으로 병목 제거 → 리랭커·캐시·gating으로 지연 관리 → 생성 품질은 RAGAS로 분리해 개선.
